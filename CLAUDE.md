## 작업 방식
- bash 명령어 직접 실행 금지
- 대신 실행해야 할 명령어를 알려주고, 내가 직접 실행 후 결과를 붙여넣으면 이어서 작업할 것
- 단, 코드 파일 읽기/수정은 직접 해도 됨
- 작업 완료 후 불필요한 요약/설명 생략할 것
- 코드 수정 시 변경된 부분만 보여줄 것 (전체 파일 출력 금지)
- 진행상황 보고는 한 줄로만
- 여러 방법 나열하지 말고 최선의 방법 하나만 제시
- 큰 작업은 시작 전에 계획을 먼저 텍스트로 보여주고 승인 받을 것
- 파일 전체를 읽지 말고 필요한 부분만 읽을 것 (grep, head 등 활용)
- 확실하지 않은 것은 추측해서 진행하지 말고 먼저 질문할 것
- 단순 반복 작업은 가장 저렴한 모델 사용
